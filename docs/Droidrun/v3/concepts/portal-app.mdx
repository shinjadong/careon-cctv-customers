> 📖 **Portal App이란?**
> 
> **비유**: 통역사나 중개인 같은 역할입니다.
> 
> - **한쪽**: 컴퓨터에서 실행되는 DroidRun (명령을 내리는 측)
> - **다른 쪽**: Android 기기 (명령을 실행하는 측)
> - **Portal App**: 양쪽의 언어를 번역하고 전달하는 중개자
> 
> 컴퓨터는 "버튼 클릭해"라고 말하고, Portal App은 Android 기기에서 실제로 그 버튼을 찾아서 클릭합니다.

## DroidRun Portal App이란?

DroidRun Portal App은 다음과 같은 작업을 수행하는 Android 애플리케이션입니다:

1. Android 기기의 **현재 UI 상태를 추출**합니다
2. 이 정보를 **ADB를 통해 DroidRun 프레임워크에 전달**합니다
3. DroidRun 프레임워크에서 보낸 **명령을 실행**합니다
4. 식별된 UI 요소에 대한 **시각적 피드백을 제공**합니다

## 작동 원리

Portal App은 [Android의 접근성 서비스(Accessibility Services)](https://developer.android.com/reference/android/accessibilityservice/AccessibilityService)를 활용하여 다음을 수행합니다:
- 기기의 UI 변경 사항 모니터링
- 상호작용 가능한 요소(버튼, 텍스트 필드 등) 식별
- 텍스트, 요소 위치 및 기타 UI 메타데이터 추출

> 📖 **Android Accessibility Services란?**
> 
> **원래 목적**: 시각/청각 장애인을 위한 스크린 리더, 음성 제어 등을 구현하기 위한 Android 시스템 기능
> 
> **DroidRun의 활용**: 이 기능이 화면의 모든 요소를 "읽을" 수 있다는 점을 활용하여, LLM 에이전트가 화면을 이해하고 조작할 수 있도록 합니다.
> 
> **실제 예시**:
> - "로그인" 버튼의 위치 파악
> - 입력 필드에 어떤 텍스트가 있는지 확인
> - 현재 화면에 어떤 앱이 열려있는지 감지

이 정보는 ADB(Android Debug Bridge)를 통해 컴퓨터에서 실행 중인 DroidRun 프레임워크로 전달되어, LLM 에이전트가 화면에 무엇이 있는지 이해하고 작업을 완료하는 방법에 대해 정보에 기반한 결정을 내릴 수 있게 합니다.

> 📖 **ADB (Android Debug Bridge)란?**
> 
> **정의**: 컴퓨터와 Android 기기를 연결하는 개발자 도구
> 
> **쉬운 비유**: USB 케이블이 물리적 연결이라면, ADB는 그 케이블을 통해 대화할 수 있게 해주는 "통신 프로토콜"입니다.
> 
> **할 수 있는 것**:
> - 명령어 전달 (예: "이 앱 실행해")
> - 파일 전송
> - 화면 정보 받아오기
> - 앱 설치/삭제
> 
> **DroidRun에서의 역할**: Portal App이 수집한 화면 정보를 컴퓨터로 보내고, 컴퓨터의 명령을 Android로 전달하는 통로

### 요소 강조 표시 (Element Highlighting)

감지된 UI 요소의 시각적 강조 표시를 활성화/비활성화할 수 있습니다:
- **사각형 오버레이(Rectangle Overlay)**: UI 요소 주위에 색상이 있는 사각형을 그립니다
- **요소 레이블(Element Labels)**: 요소 ID 또는 설명을 표시합니다

> 💡 **실무 활용 팁**
> 
> 이러한 시각적 표시는 다음을 이해하는 데 도움이 됩니다:
> 
> 1. **디버깅**: DroidRun이 어떤 요소를 "볼 수 있는지" 확인
>    - 예: "왜 버튼을 못 찾지?" → 오버레이로 확인하면 버튼이 감지 안 되는 걸 확인 가능
> 
> 2. **요소 분류 확인**: 요소가 어떻게 분류되고 있는지 파악
>    - 예: 이미지 버튼이 "이미지"로만 인식되는지, "버튼"으로도 인식되는지
> 
> 3. **LLM에게 전달되는 정보 검증**: LLM 에이전트가 받는 정보가 정확한지 확인
>    - 예: "로그인"이라는 텍스트가 제대로 추출되는지

## 개인정보 보호 및 보안

DroidRun Portal App은:
- **오직 ADB를 통해서만** DroidRun 프레임워크와 통신합니다
- **외부 서버로 데이터를 전송하지 않습니다**
- DroidRun 프레임워크가 사용할 때**만 활성화**됩니다
- 사용하지 않을 때는 Android 설정에서 **쉽게 비활성화**할 수 있습니다

> 🔒 **보안 관련 중요 사항**
> 
> - **로컬 전용**: 모든 통신이 내 컴퓨터 ↔ 내 기기 사이에서만 발생
> - **인터넷 불필요**: 인터넷 연결 없이도 작동 (LLM 호출 제외)
> - **권한 제어 가능**: Android 설정에서 언제든 접근성 권한 해제 가능
> - **오픈소스**: 코드가 공개되어 있어 보안 검증 가능

## 설치

DroidRun Portal App은 [DroidRun Portal 저장소](https://github.com/droidrun/droidrun-portal)에서 다운로드할 수 있습니다. 설치 방법은 [빠른 시작 가이드(Quickstart)](/v3/quickstart)를 참조하세요.

> 💡 **설치 전 확인사항**
> 
> 1. **개발자 옵션 활성화**: Android 설정에서 개발자 모드 켜기
> 2. **USB 디버깅 허용**: ADB 연결을 위해 필수
> 3. **접근성 권한 부여**: Portal App이 화면 정보를 읽을 수 있도록 허용
> 
> 자세한 단계별 가이드는 빠른 시작 문서에 포함되어 있습니다.
