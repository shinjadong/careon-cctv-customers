# 텔레메트리가 필요한 이유

텔레메트리는 다음과 같은 목적으로 활용됩니다:
- 어떤 기능이 가장 많이 사용되고 개선이 필요한지 파악
- 실제 사용 패턴에 기반하여 버그 수정 및 신규 기능 우선순위 결정
- DroidRun이 다양한 환경에서 잘 작동하는지 확인

> 📖 **텔레메트리(Telemetry)란?**
> 
> **정의**: 소프트웨어 사용 현황을 원격으로 수집하는 기술
> 
> **쉬운 비유**: 자동차 제조사가 차량의 주행 데이터(평균 속도, 브레이크 사용 빈도 등)를 수집해서 다음 모델 개선에 활용하는 것과 같습니다.
> 
> **DroidRun에서 수집하는 정보 (예상)**:
> - ✅ 어떤 명령어가 자주 사용되는지
> - ✅ 어떤 LLM 프로바이더가 인기 있는지
> - ✅ 오류가 발생하는 시나리오
> - ✅ 평균 작업 완료 시간
> 
> **수집하지 않는 정보**:
> - ❌ 개인 식별 정보 (이름, 이메일 등)
> - ❌ 실행한 구체적인 명령 내용
> - ❌ 기기 정보 (IMEI, 시리얼 번호 등)
> - ❌ 스크린샷이나 화면 내용

<b>개인정보나 민감한 데이터는 일체 수집하지 않습니다.</b> 모든 텔레메트리는 엄격히 익명화되어 있으며, 오직 모든 사용자를 위해 프레임워크를 개선하는 용도로만 사용됩니다.

> 🔒 **익명화(Anonymized)의 의미**
> 
> **정의**: 개인을 특정할 수 없도록 데이터를 처리하는 것
> 
> **실제 예시**:
> - ❌ 나쁜 예: "사용자 john@example.com이 '카카오톡 열어줘' 명령 실행"
> - ✅ 좋은 예: "익명 사용자가 '앱 실행' 카테고리 명령 실행"
> 
> **익명화 기법**:
> - 사용자 고유 ID 제거 또는 해시 처리
> - 명령 내용을 카테고리로 일반화
> - IP 주소 마스킹
> - 타임스탬프 일반화 (정확한 시간 대신 시간대)

질문이나 우려사항이 있으시면 [GitHub](https://github.com/droidrun/droidrun)에서 문의하거나 개인정보 보호 정책을 검토해 주세요.

# 텔레메트리 설정하기
DroidRun은 <b>익명화된</b> 사용 데이터를 수집하여 어떤 기능이 가장 가치 있고 어디에 개선이 필요한지 파악하는 데 도움을 받습니다. 이 데이터는 <b>절대로</b> 광고나 개인 추적 목적으로 사용되지 않으며, 오직 커뮤니티를 위해 DroidRun을 더 나은 도구로 만드는 데만 사용됩니다.
## 텔레메트리 비활성화하는 방법

다음 환경 변수를 설정하여 언제든지 텔레메트리를 비활성화할 수 있습니다:

```bash
export DROIDRUN_TELEMETRY_ENABLED=false
```

> 💡 **환경 변수(Environment Variable) 설정 가이드**
> 
> **환경 변수란?**
> - 운영체제 전체에서 사용되는 설정 값
> - 프로그램이 시작할 때 읽어서 동작 방식 결정
> 
> **쉬운 비유**: 집 전체에 적용되는 온도 설정
> - 개별 방이 아닌 전체 집의 난방 온도를 정하는 것
> - 한 번 설정하면 모든 방(프로그램)에 적용됨

이 라인을 shell profile 파일(예: `.bashrc`, `.zshrc`, 또는 `.profile`)에 추가하면 세션 간에도 설정이 유지됩니다.

> 🔧 **Shell Profile 파일 선택 가이드**
> 
> **어떤 파일에 추가해야 할까?**
> 
> | Shell | 파일 위치 | 사용 시나리오 |
> |-------|----------|--------------|
> | **Bash** | `~/.bashrc` | Linux 기본 쉘, Ubuntu/Debian |
> | **Zsh** | `~/.zshrc` | macOS 기본 쉘 (Catalina 이후) |
> | **모든 쉘** | `~/.profile` | 로그인 시 한 번만 실행 |
> 
> **내가 어떤 쉘을 쓰는지 확인하기**:
> ```bash
> echo $SHELL
> # 출력 예시:
> # /bin/bash  → .bashrc 사용
> # /bin/zsh   → .zshrc 사용
> ```
> 
> **설정 파일에 추가하는 방법 (Windows 제외)**:
> ```bash
> # 1. 파일 열기
> nano ~/.zshrc              # Mac (Zsh)
> nano ~/.bashrc             # Linux (Bash)
> 
> # 2. 파일 끝에 아래 라인 추가
> export DROIDRUN_TELEMETRY_ENABLED=false
> 
> # 3. 저장 (Ctrl+O, Enter, Ctrl+X)
> 
> # 4. 설정 즉시 적용
> source ~/.zshrc            # Mac
> source ~/.bashrc           # Linux
> ```
> 
> **Windows (PowerShell)**:
> ```powershell
> # 1. PowerShell 프로필 열기
> notepad $PROFILE
> 
> # 2. 아래 라인 추가
> $env:DROIDRUN_TELEMETRY_ENABLED = "false"
> 
> # 3. 저장 후 PowerShell 재시작
> ```
> 
> **Windows (환경 변수 GUI 설정)**:
> 1. 시스템 속성 → 고급 → 환경 변수
> 2. 사용자 변수에서 "새로 만들기"
> 3. 변수 이름: `DROIDRUN_TELEMETRY_ENABLED`
> 4. 변수 값: `false`

## 텔레메트리 다시 활성화하는 방법

텔레메트리를 다시 활성화하려면 환경 변수를 `true`로 설정하세요:

```bash
export DROIDRUN_TELEMETRY_ENABLED=true
```

또는 환경에서 변수를 제거하여 기본 동작(활성화됨)을 사용하면 됩니다.

> 💡 **텔레메트리 상태 확인하기**
> 
> **현재 설정 확인**:
> ```bash
> echo $DROIDRUN_TELEMETRY_ENABLED
> # 출력:
> # false → 비활성화됨
> # true 또는 빈 값 → 활성화됨 (기본값)
> ```
> 
> **환경 변수 삭제하기 (기본값으로 복원)**:
> ```bash
> unset DROIDRUN_TELEMETRY_ENABLED
> ```
> 
> **Shell profile 파일에서 제거**:
> 1. `~/.zshrc` 또는 `~/.bashrc` 열기
> 2. `export DROIDRUN_TELEMETRY_ENABLED=false` 라인 삭제
> 3. 저장 후 `source ~/.zshrc` 실행

## 자주 묻는 질문 (FAQ)

> ❓ **텔레메트리를 비활성화하면 기능이 제한되나요?**
> 
> 아니요, 모든 기능이 정상적으로 작동합니다. 단지 사용 통계가 수집되지 않을 뿐입니다.

> ❓ **정말 개인정보가 수집되지 않나요?**
> 
> DroidRun은 오픈소스입니다. [GitHub 코드](https://github.com/droidrun/droidrun)를 직접 검토하여 무엇이 수집되는지 확인할 수 있습니다.

> ❓ **텔레메트리를 활성화하면 어떤 도움이 되나요?**
> 
> - 자주 사용되는 기능 파악 → 버그 수정 우선순위 결정
> - 다양한 환경(기기, OS, LLM) 테스트 데이터 확보
> - 실제 사용 패턴 기반의 개선 방향 설정

> ❓ **회사/조직에서 사용 시 텔레메트리를 꺼야 하나요?**
> 
> 조직의 보안 정책에 따라 다릅니다:
> - 익명화된 통계도 외부 전송 금지 → 비활성화 권장
> - 개인정보만 금지 → 활성화 가능 (개인정보 미수집)
> - IT 부서와 상의 권장
